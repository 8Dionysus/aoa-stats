# method-growth mechanic

`mechanics/method-growth/` is the active aoa-stats implementation of the common
`method-growth` mechanic.

Observes candidate lineage and explicit supersession without deciding
promotion. Owner-landing receipt kinds remain shared turnover inputs, but their
example-only standalone summary and mechanic part are retired.

Start with `PARTS.md`, then use the selected part's `README.md`,
`CONTRACT.md`, and `VALIDATION.md`.

The two active parts share deterministic projection code in
`src/aoa_stats_builder/candidate_lifecycle.py`, but each owns its focused proof
under its own `tests/` district. The root `scripts/build_views.py` only exposes
that core through repo-wide build and compatibility orchestration.
