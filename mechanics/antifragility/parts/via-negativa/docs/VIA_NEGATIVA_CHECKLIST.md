# VIA_NEGATIVA_CHECKLIST

This checklist is for `aoa-stats` as the derived observability and
machine-first summaries.

## Keep intact

- source evidence that a stronger owner must retain
- derived summaries with clear consumer value
- suppression semantics for low evidence
- vector and window views that preserve signal shape

## Route check

Try the pruning order: keep, merge, move, suppress, quarantine, deprecate,
delete. For every candidate route, name the owner and the provenance that
survives.

Candidates include:

- scalar prestige views
- summaries with no real reader or consumer
- derived docs that sound canonical rather than derived

## Questions before adding anything new

1. Who consumes this summary and why?
2. Can a vector survive where a score would distort?
3. Should this view suppress itself when evidence is thin?
4. Which owner must review a move or deletion?
5. Which source reference or retained lesson preserves provenance?

## Stop lines

- Do not delete or move source evidence that must be retained.
- Do not move or delete a surface before its owner repository reviews the
  candidate.
- Route proof failure first to `aoa-evals`; subtraction cannot repair missing
  proof.
- Treat the result as a review candidate or owner-local handoff, never as
  authority to mutate a surface.
- Checklist presence does not prove that a review occurred.

## Safe exceptions

- a new derived view that unlocks a clear bounded consumer workflow
- short bridge docs needed to migrate away from bad scalar habits

## Exit condition

- Stats stays derived, useful, and humble; the owner and surviving provenance
  are explicit for every proposed subtraction.
