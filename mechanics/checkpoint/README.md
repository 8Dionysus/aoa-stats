# checkpoint mechanic

`mechanics/checkpoint/` is the active aoa-stats implementation of the common
`checkpoint` mechanic.

Summarizes bounded closeout and handoff posture while stronger runtime owners retain gate truth.

The active stats operation is a committed compatibility projection of the
historical `runtime_wave_closeout_receipt`. Current `abyss-stack` trial code
publishes a differently named receipt and feed, so this package does not claim
that the compatibility summary is current live runtime state.

Start with `PARTS.md`, then use the selected part's `README.md`,
`CONTRACT.md`, and `VALIDATION.md`.
