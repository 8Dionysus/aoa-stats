# Derived signal hygiene

## Purpose

This guide keeps a consumer from turning several weaker derived signals into a
closed loop that appears stronger than the source evidence each signal depends
on. It is part of the Boundary Bridge consumer-regrounding operation, not an
independent proof or routing authority.

## Authority order

Read and interpret the chain in this direction:

1. source-owned receipts and owner-local artifacts
2. bounded eval reports
3. derived stats summaries
4. derived route hints
5. memo recall objects

A later item may point back to an earlier one or help choose the next
inspection. It cannot validate, repair, or promote an earlier item.

## Consumer procedure

1. Name the source owner and the exact fact or receipt being summarized.
2. Inspect the bounded proof surface when the claim depends on quality.
3. Read the stats surface only for its declared question and authority ceiling.
4. Treat a route hint as navigation, not evidence that the route succeeded.
5. Treat recalled memory as context, not proof that the present case matches.
6. Return to the strongest missing owner surface before action or mutation.

## Safe interpretation

- A summary may describe counts, posture, or movement inside its declared
  window; it does not authorize mutation.
- A route hint may select a bounded inspection path; it does not prove recovery
  or owner health.
- A memo object may recall a prior pattern; it does not establish present
  identity, recurrence, or causality.
- Agreement among stats, routing, and memory remains correlated derived
  evidence unless an owner fact and required bounded proof support the claim.

## Return-to-owner triggers

Reground immediately when:

- the source owner or evidence ref is missing
- a derived surface cites another derived surface as its only support
- a route or memory signal is being used as a verdict
- stale or example-backed input is being read as current state
- a consumer is about to mutate an owner system from summary posture alone

## Stop lines

- no closed citation loop becomes proof
- no derived majority vote overrides owner evidence
- no missing proof is inferred from agreement between projections
- no access plane, cache, catalog, or MCP response changes the authority order

The stable repository-wide authority ladder remains in
`docs/ARCHITECTURE.md`. Exact surface inputs and ceilings remain in authored
profiles and the catalog; this guide owns only the consumer operation that
applies those limits across the bridge.
