# AGENTS.md

## Agon parts guide

Each child directory is one active, bounded stats operation. Before changing a
part, read the package `AGENTS.md` and `PARTS.md`, then the part's `README.md`,
`CONTRACT.md`, and `VALIDATION.md`.

Keep its source payload local. A builder may cross the package boundary only to
publish the root generated output named in its contract. Preserve stronger
owner truth, candidate-only posture, and every declared stop line.

Do not add a part merely because a file contains the word `agon`; require a
cohesive operation, focused validation, and a clear output contract.
