# Titan Incarnation Metrics

The committed Incarnation surface derives only three counts after validating
the checked-in owner examples:

- `seeded_titans`: identities present in all three examples
- `default_active`: identities that all three examples classify as active
- `locked_by_gate`: non-active identities with the same non-empty gate in all
  three examples

The source chain is a reference example chain, not a current session. These
counts do not prove incarnation, summon, gate passage, or runtime activity.
Metrics are pulse, not verdict.
